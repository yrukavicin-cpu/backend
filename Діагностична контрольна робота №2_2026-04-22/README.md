## Рівень 1

1. **Поясніть на конкретному прикладі, яким чином можна організувати додавання об'єктів?**  
   Додавання об'єктів можна організувати через перевантаження оператора `+`. Наприклад, для класу `Vector2D` можна додавати координати двох векторів:

   ```cpp
   Vector2D operator+(const Vector2D& other) {
       return Vector2D(x + other.x, y + other.y);
   }
   ```

2. **Чи особливий випадок додавання "число + об'єкт"? Наведіть конкретний приклад.**  
   Так, бо лівим операндом є число, а не об'єкт класу. Тому оператор часто роблять зовнішньою функцією.

   ```cpp
   Vector2D operator+(int value, const Vector2D& v) {
       return Vector2D(v.x + value, v.y + value);
   }
   ```

3. **Як здійснюється перевантаження оператора `[]`? Наведіть конкретний приклад.**  
   Оператор `[]` перевантажують як метод класу. Він дозволяє звертатися до елементів об'єкта як до масиву.

   ```cpp
   int& operator[](int index) {
       return data[index];
   }
   ```

4. **Які можливості перевантаження операторів в інших мовах програмування?**  
   У C++ перевантаження операторів підтримується широко. У C# теж можна перевантажувати частину операторів. У Java звичайні оператори для власних класів перевантажувати не можна, крім спеціальної поведінки `+` для рядків.

5. **Які ключові слова можна використати в реалізації Visual C++ для оголошення класів-інтерфейсів? Наведіть конкретний приклад.**  
   У Visual C++ можна використовувати `interface` або `__interface`. Такий клас містить лише оголошення методів.

   ```cpp
   __interface IPrintable {
       void print();
   };
   ```

6. **Чому не є коректним повернення функціями вказівників на локальні об'єкти? Поясніть відповідь на конкретному прикладі.**  
   Локальний об'єкт знищується після завершення функції. Тому вказівник на нього буде вказувати на вже неіснуючу область пам'яті.

   ```cpp
   int* getValue() {
       int x = 10;
       return &x; // помилка
   }
   ```

7. **Що не повинно бути у класі-інтерфейсі згідно з канонами ООП? Поясніть відповідь на конкретному прикладі.**  
   У класі-інтерфейсі не повинно бути полів даних і готової реалізації логіки. Він має описувати лише набір методів.

   ```cpp
   class IShape {
   public:
       virtual double area() const = 0;
       virtual ~IShape() = default;
   };
   ```

8. **Чи може бути в класі-інтерфейсі віртуальний деструктор? Поясніть відповідь на конкретному прикладі.**  
   Так, і це правильно. Віртуальний деструктор потрібен, щоб об'єкт похідного класу коректно знищувався через вказівник на інтерфейс.

   ```cpp
   IShape* shape = new Circle();
   delete shape; // викликається деструктор Circle
   ```

## Рівень 2

1. **Що потрібно передбачити у класі задля забезпечення можливості передавання його об'єктів у якості параметра функції? Наведіть конкретний приклад.**  
   Потрібно передбачити коректне копіювання або передавати об'єкт за посиланням. Найчастіше використовують `const`-посилання, щоб не створювати зайву копію.

   ```cpp
   void print(const Student& student) {
       student.show();
   }
   ```

2. **У чому полягає відмінність класів-інтерфейсів від абстрактних класів? Наведіть конкретний приклад програмної реалізації.**  
   Клас-інтерфейс містить тільки набір методів без стану. Абстрактний клас може мати поля, конструктор і частково реалізовані методи.

   ```cpp
   class IPrintable {
   public:
       virtual void print() const = 0;
       virtual ~IPrintable() = default;
   };

   class Shape {
   protected:
       std::string color;

   public:
       virtual double area() const = 0;
       void setColor(const std::string& value) { color = value; }
       virtual ~Shape() = default;
   };
   ```

3. **Як запрограмувати двовимірний масив на основі класу `vector`? Наведіть конкретний приклад програмної реалізації.**  
   Можна створити `vector`, елементами якого є інші `vector`.

   ```cpp
   std::vector<std::vector<int>> matrix(3, std::vector<int>(4, 0));

   matrix[0][1] = 5;
   ```

4. **Чим відрізняється алгоритм `count_if` від `find_if`? Наведіть конкретний приклад програмної реалізації.**  
   `count_if` рахує кількість елементів, які відповідають умові. `find_if` знаходить перший такий елемент.

   ```cpp
   auto isEven = [](int x) { return x % 2 == 0; };

   int count = std::count_if(v.begin(), v.end(), isEven);
   auto first = std::find_if(v.begin(), v.end(), isEven);
   ```

5. **Що потрібно передбачити, щоб функції могли повертати об'єкти у якості результату своєї роботи? Наведіть конкретний приклад програмної реалізації.**  
   Клас повинен мати коректний конструктор і можливість копіювання або переміщення. У сучасному C++ об'єкти можна безпечно повертати за значенням.

   ```cpp
   Student createStudent() {
       return Student("Ivan", 18);
   }
   ```

6. **Чим контейнер `list` відрізняється від `vector`? Здійсніть порівняльну характеристику на конкретних прикладах.**  
   `vector` зберігає елементи в суцільній області пам'яті, тому швидко працює з індексами. `list` є двозв'язним списком, тому зручний для частих вставок і видалень у середині.

   ```cpp
   std::vector<int> v = {1, 2, 3};
   std::cout << v[1]; // швидкий доступ за індексом

   std::list<int> l = {1, 2, 3};
   l.push_front(0); // швидка вставка на початок
   ```

7. **Поясніть на конкретному прикладі програмної реалізації, що робить алгоритм `for_each`. Навіщо у контейнерах `map` зберігаються пари?**  
   `for_each` виконує задану дію для кожного елемента діапазону.

   ```cpp
   std::for_each(v.begin(), v.end(), [](int x) {
       std::cout << x << " ";
   });
   ```

   У `map` зберігаються пари `ключ-значення`, бо доступ до значення відбувається за ключем.

   ```cpp
   std::map<std::string, int> ages;
   ages["Ivan"] = 18;
   ```

8. **Проаналізуйте можливі варіанти виконання операцій з об'єктами та заповніть таблицю.**

   | Дія | Запис | Конструктор звичайний | Конструктор копії | Рекомендації щодо визначення/перевантаження |
   | --- | --- | --- | --- | --- |
   | Створення об'єкта | `Student a;` | Викликається | Ні | Потрібен звичайний конструктор |
   | Створення з параметрами | `Student a("Ivan", 18);` | Викликається | Ні | Варто визначити конструктор з параметрами |
   | Копіювання | `Student b = a;` | Ні | Викликається | Потрібен конструктор копії, якщо є динамічна пам'ять |
   | Передавання у функцію | `print(a);` | Ні | Може викликатись | Краще передавати як `const Student&` |
   | Повернення з функції | `Student a = createStudent();` | Може викликатись | Може оптимізуватись | Можна повертати за значенням |
   | Присвоювання | `b = a;` | Ні | Ні | Потрібно перевантажити `operator=`, якщо є ресурси |
