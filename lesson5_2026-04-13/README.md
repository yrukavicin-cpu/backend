# Lesson 5, 2026-04-13

## Тема
Питання для самостійного опрацювання: параметричний поліморфізм і шаблони.

## Відповіді
1. **Поясніть зміст термінів узагальнене програмування, параметризований клас.**  
   Узагальнене програмування — це підхід, коли код пишеться не під один конкретний тип, а більш універсально. Параметризований клас — це клас, у який тип або значення передається як параметр.

2. **Наведіть приклад опису шаблонного класу Точка в N-вимірному просторі.**  
   Приклад: `template <class T, int N> class Point { T coord[N]; };`  
   Тут `T` — тип координат, а `N` — кількість вимірів.

3. **Які типи можна використовувати як фактичний параметр-тип?**  
   Можна використовувати стандартні типи, класи, структури, покажчики та інші типи, які підходять під умови шаблону.

4. **Чи існують обмеження на класи, які можуть бути використанні як фактичні параметри при конкретному інстанціюванні шаблону?**  
   Так. Тип повинен підтримувати ті операції, які використовуються в шаблоні.

5. **Чим відрізняється використання звичайного класу і інстанційованого? Шаблонного і нешаблонного класу?**  
   Звичайний клас уже має конкретний тип. Шаблонний клас є загальним описом. Інстанційований клас — це вже готовий варіант шаблону для конкретного типу, наприклад `vector<int>`.

6. **В яких випадках статичні члени шаблонного класу дублюються для кожного інстанціювання?**  
   Для кожної конкретної версії шаблону створюється свій статичний член. Наприклад, `Box<int>` і `Box<double>` матимуть різні статичні поля.

7. **Чи може шаблон бути похідним від звичайного класу? Від шаблону? В яких випадках від шаблонного може походити звичайний клас?**  
   Так, шаблон може успадковуватись і від звичайного класу, і від іншого шаблону. Звичайний клас теж може наслідуватись від шаблонного, якщо вказати конкретні параметри.

8. **На якому етапі виконання програми створюється виконуваний код для версії шаблону класу? Для функції – члена шаблону.**  
   Код для конкретного шаблону створюється під час компіляції.

9. **В чому полягають переваги і недоліки використання об’єктів типу string.**  
   Переваги: зручно працювати з рядками, пам’ять керується автоматично, є багато готових методів. Недоліки: іноді повільніше за `char`-масив і менше контролю на низькому рівні.

10. **Шаблонний клас string і його методи для роботи з рядками символів.**  
    `basic_string` — це шаблонний клас для роботи з рядками. Основні методи: `size()`, `length()`, `empty()`, `append()`, `substr()`, `find()`, `erase()`, `insert()`, `replace()`.

11. **Шаблонний клас vector і його методи для роботи з динамічними векторами.**  
    `vector` — це шаблонний клас динамічного масиву. Основні методи: `push_back()`, `pop_back()`, `size()`, `empty()`, `clear()`, `at()`, `front()`, `back()`, `insert()`, `erase()`, `resize()`.

## Висновок
У роботі коротко розглянуто шаблони, узагальнене програмування та стандартні класи `string` і `vector`.
