# Lesson 11, 2026-04-13

## Тема
Оброблення виняткових ситуацій. Різновиди патернів Factory.

## Відповіді
1. **Що виконує фабричний метод у класі?**  
   Фабричний метод створює об'єкти потрібного типу і повертає їх клієнтському коду. Його головна задача — винести створення об'єкта в окремий метод.

2. **Навіщо може бути потрібен фабричний метод у деякому класі? Які проблеми можна вирішити за його допомогою?**  
   Фабричний метод потрібен для того, щоб не створювати об'єкти напряму через `new` у багатьох місцях програми. Так простіше змінювати тип створюваних об'єктів, зменшувати залежність між класами і робити код більш гнучким.

3. **Яка структура патерну Фабричний метод?**  
   Зазвичай є базовий клас або інтерфейс продукту, конкретні класи продуктів, базовий клас творця з фабричним методом і похідні класи творця, які перевизначають цей метод.

4. **У чому зручність для клієнта замість безпосереднього створення потрібних об'єктів створювати для цього фабрику?**  
   Клієнту не треба знати, який саме конкретний клас створюється. Він працює через загальний інтерфейс, а вся логіка створення ховається у фабриці.

5. **Що може приховувати фабрика класів від клієнта?**  
   Фабрика може приховувати конкретні класи, умови створення об'єктів, налаштування, порядок ініціалізації і взагалі всю внутрішню логіку створення.

6. **Чому патерн Фабричний метод інколи звуть віртуальним конструктором?**  
   Тому що він дозволяє створювати об'єкти в похідних класах через спільний інтерфейс, схоже на те, як конструктор створює об'єкт. Але при цьому вибір конкретного типу може відбуватись поліморфно.

7. **Які відмінності патерну "Абстрактна фабрика" від патерну "Фабричний метод"?**  
   `Фабричний метод` зазвичай відповідає за створення одного продукту через перевизначення методу в похідних класах.  
   `Абстрактна фабрика` створює цілу групу пов'язаних об'єктів і зазвичай містить кілька фабричних методів.
