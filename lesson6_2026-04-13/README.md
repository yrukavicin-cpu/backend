# Lesson 6, 2026-04-13

## Тема
Параметричний поліморфізм. Контейнери.

## Відповіді
1. **Які складові стандартної бібліотеки C++?**  
   Основні складові стандартної бібліотеки C++:
   - контейнери (`vector`, `list`, `map`);
   - алгоритми (`sort`, `find`, `copy`);
   - ітератори;
   - рядки (`string`);
   - потоки введення і виведення (`iostream`);
   - функціональні об'єкти та математичні функції.

2. **Чи можливо в програмі на C++ використати бібліотеку C?**  
   Так, можливо. У C++ можна використовувати бібліотеки C. Зазвичай для цього підключають заголовки типу `<cmath>`, `<cstdlib>`, `<cstdio>`.

   Приклад:
   ```cpp
   #include <cmath>
   double x = sqrt(16.0);
   ```

3. **Чим відрізняється клас `complex` від `pair`?**  
   `std::complex` використовується для комплексних чисел і зберігає дійсну та уявну частини.  
   `std::pair` — це просто пара двох значень, які можуть бути навіть різних типів.

   Приклад:
   ```cpp
   std::complex<double> c(3.0, 4.0);
   std::pair<int, std::string> p(1, "Apple");
   ```

4. **Які типи символів підтримує клас `string`?**  
   Є такі основні варіанти:
   - `char` — `std::string`;
   - `wchar_t` — `std::wstring`;
   - `char16_t`, `char32_t` — для Unicode.

5. **Що таке конкатенація?**  
   Конкатенація — це об'єднання кількох рядків в один.

   Приклад:
   ```cpp
   std::string a = "Hello";
   std::string b = " World";
   std::string c = a + b;
   ```

6. **Які відмінності `size` від `capacity` в класі `vector`?**  
   `size()` — це скільки елементів зараз є у векторі.  
   `capacity()` — це скільки елементів вектор може вмістити без нового виділення пам'яті.

   Приклад:
   ```cpp
   std::vector<int> v;
   v.reserve(100);
   v.push_back(5);
   ```
   Тут `size = 1`, а `capacity = 100`.

7. **Як запрограмувати двовимірний масив на основі класу `vector`?**  
   Для цього використовують вектор векторів.

   Приклад:
   ```cpp
   std::vector<std::vector<int>> matrix(3, std::vector<int>(3, 0));
   matrix[0][1] = 5;
   ```

8. **Які недоліки у контейнерів типу `vector`?**  
   Основні недоліки:
   - повільна вставка і видалення в середині або на початку;
   - при нестачі пам'яті може відбуватись перевиділення і копіювання елементів.

9. **Чим контейнер `list` відрізняється від `vector`?**  
   `vector` зберігає елементи підряд у пам'яті й підтримує доступ за індексом.  
   `list` є двонаправленим списком, не підтримує доступ за індексом, але зручний для вставки і видалення елементів.

10. **Що таке асоціативний контейнер?**  
    Це контейнер, у якому дані зберігаються за ключами. Наприклад, `map` і `set`.

11. **Навіщо у контейнерах `map` зберігаються пари?**  
    Тому що `map` працює за принципом "ключ-значення". Кожен елемент — це пара: ключ і відповідне значення.

    Приклад:
    ```cpp
    std::map<std::string, int> ages;
    ages.insert(std::make_pair("Іван", 19));
    ```

12. **Що означає суфікс `_if`?**  
    Суфікс `_if` означає, що алгоритм працює не з конкретним значенням, а з умовою.

    Приклад:
    `find_if`, `count_if`, `remove_if`.

13. **Що таке ітератор? Наведіть приклади.**  
    Ітератор — це об'єкт, який дозволяє проходити по елементах контейнера.

    Приклад:
    ```cpp
    std::vector<int> v = {1, 2, 3};
    for (std::vector<int>::iterator it = v.begin(); it != v.end(); ++it) {
        std::cout << *it << " ";
    }
    ```

14. **Що таке функціональний об'єкт?**  
    Функціональний об'єкт або функтор — це об'єкт, у якого перевантажений оператор `()`. Його можна викликати як функцію.

    Приклад:
    ```cpp
    struct IsEven {
        bool operator()(int n) { return n % 2 == 0; }
    };
    ```

15. **Чим відрізняється алгоритм `count_if` від `find_if`?**  
    `count_if` рахує, скільки елементів задовольняють умову.  
    `find_if` знаходить перший елемент, який задовольняє умову.

16. **Що робить алгоритм `for_each`?**  
    `for_each` застосовує функцію або функтор до кожного елемента в заданому діапазоні.

    Приклад:
    ```cpp
    void print(int n) { std::cout << n << " "; }
    std::vector<int> v = {1, 2, 3};
    std::for_each(v.begin(), v.end(), print);
    ```
