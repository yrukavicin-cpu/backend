1. Проаналізуйте один з видів реалізації статичного поліморфізму - перевантаження функцій

Статичний поліморфізм = вибір функції на етапі компіляції.
Перевантаження — це коли кілька функцій мають однакове ім’я, але різні параметри.

Одна і та сама функція може робити схожі дії з різними типами/кількістю параметрів. Компілятор сам вибирає, яку викликати.
Приклад:

int sum(int a, int b) { return a + b; }
double sum(double a, double b) { return a + b; }
int sum(int a) { return a + 10; }

2. Проаналізуйте особливості застосування оператора catch(...) мови C++

catch(...) — це "ловець" виключень.
Ловить будь-які виключення, які не описали окремими catch.

Ловить абсолютно всі типи помилок
Зручно ставити останнім блоком для "всього іншого"
Не дає інформації про тип помилки
Часто використовують для фінального перехоплення, щоб програма "не падала"))

Приклад:

try {
    // код
}
catch(int) {
    // ловимо int
}
catch(...) {
    // ловимо все, що інше
}
